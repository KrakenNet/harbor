# ML Model · `clustering`

Generic clustering over arbitrary embeddings or feature vectors.
Returns cluster assignments, centroids, and a silhouette score.

## Purpose

Group similar items without labels — customer segments, ticket
clusters, document corpora, embedding-cartography. Feeds the
`embedding-cartographer` tool, the `topic-model` pipeline, and any
"find groups" UI.

## Task type

Unsupervised clustering. Default: HDBSCAN over normalized embeddings;
k-means and Gaussian mixtures available as alternatives.

## Inputs

| Field            | Type        | Notes                              |
|------------------|-------------|------------------------------------|
| `vectors`        | float[][]   | n × d input matrix                 |
| `k`              | int         | optional; ignored by HDBSCAN       |
| `min_cluster_size` | int       | for density methods                |

## Outputs

| Field          | Type        | Notes                              |
|----------------|-------------|------------------------------------|
| `labels`       | int[]       | cluster id per row, `-1` = noise   |
| `centroids`    | float[][]   | when applicable                    |
| `silhouette`   | float       | quality summary                    |
| `cluster_summaries[]` | list[ClusterSummary] | size, density, top members |

## Training-data shape

A vector matrix; usually generated by `embedding-encoder`. Demo seed:
20k document embeddings from a sample news corpus.

## Eval metric

Silhouette and Davies-Bouldin where labels are absent; ARI/NMI when
ground-truth groupings exist.

## Serving target

gomlx (`internal/gomlx/`) for k-means at scale; in-process Go for
HDBSCAN. Vector input means batching is the throughput knob.

## Why it's a good demo

Pairs naturally with `embedding-encoder` and `topic-model` to form a
"vectors → groups → labels" pipeline. Often the second model in a
pipeline that begins with embeddings, so the demo can show pipeline
composition cleanly.
